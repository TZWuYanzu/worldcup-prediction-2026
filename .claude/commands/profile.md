# 球队档案

生成指定球队的完整结构性分析档案，格式参照 `02_data/team_profiles_knockout.md`。

## 解析用户意图

从 `$ARGUMENTS` 中提取球队名：
- 中文（如 `法国`、`阿根廷`）→ 通过 `match_data.CN_TO_EN` 转为英文
- 英文（如 `France`、`Argentina`）→ 直接使用
- 多个球队（如 `法国 德国`）→ 逐一生成档案

## 执行步骤

### Step 1: xG 汇总

```bash
python3 xg_model.py team <team_name>
```

记录：每场xG/xGA、合计xGD、转化效率、分阶段xG。

### Step 2: 换人效果

```bash
python3 xg_model.py subs <team_name>
```

### Step 3: 热身赛数据

用 WebFetch 获取 Wikipedia 页面：
```
https://en.wikipedia.org/wiki/<Team>_national_football_team
```

提取 2026 年 1-6 月所有热身赛结果。

### Step 4: 综合生成档案

### 关键原则

- **xG为核心、热身赛为辅** — xG反映真实比赛质量，热身赛提供补充样本
- **区分"能力超额"和"运气超额"** — 超级巨星（Messi/Mbappé）的超额转化可能部分是能力
- **防守评估含乌龙球和点球** — 它们反映禁区压力下的抗压极限
- **对强队成绩权重更高** — 热身赛对弱队的大胜参考价值低
