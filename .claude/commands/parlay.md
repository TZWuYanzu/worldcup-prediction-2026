# 串关策略

基于最新预测和赔率，设计胜负平(HAD)或比分(CRS)串关方案。

## 解析用户意图

从 `$ARGUMENTS` 中提取：
- 无参数 → 同时分析 HAD 和 CRS 两种串关
- `had` → 仅分析胜负平串关
- `crs` / `比分` → 仅分析比分串关
- 包含 match_id（如 `59 60 65`）→ 只在这些比赛中构建串关
- `串N` 或 `N串` → 指定串关类型（如 串3、4串3）

## 执行步骤

### Step 1: 获取最新赔率和预测

```bash

```

### Step 2: 筛选候选场次

**HAD 串关筛选标准：**
- 模型概率 vs 赔率隐含概率有 edge（>5%）
- 排除赔率极低（<1.30）的比赛——低赔率串关压缩回报
- 优先选择方向明确（最高概率 >50%）的比赛

**CRS 比分串关筛选标准：**
- 选择方向确定性高的比赛（预测比分概率排名前2-3位）
- 每场选 1-3 个比分覆盖
- 需要平衡覆盖面和注数

### Step 3: 组合计算

**注数计算公式：**
- N 场比赛，每场分别选 a₁, a₂, ..., aₙ 个选项
- M串N（从M场中选N场）: C(M,N) 种组合
- 每种组合的注数 = 所选各场选项数之积
- 总注数 = Σ 各组合注数
- 总金额 = 总注数 × 2元/注

**示例计算：**
4场比赛选项数为(2,1,3,1)，做4串3：
- C(4,3) = 4 种组合
- 组合1: 2×1×3 = 6注
- 组合2: 2×1×1 = 2注
- 组合3: 2×3×1 = 6注
- 组合4: 1×3×1 = 3注
- 总计: 17注 × 2 = 34元

### Step 4: 回报分析

计算每种命中情景的预期回报：
- 最低回报（全选最低赔率命中）
- 最高回报（全选最高赔率命中）
- 加权预期回报

### Step 5: CRS 每场双比分分析

当分析 CRS 串关时，**始终额外输出每场选2个比分的方案**。

**选分逻辑（每场选2个比分）：**

1. 从 `02_data/sporttery_crs.json` 读取所有比分赔率
2. 用模型概率（来自 score_matrix.py）计算每个比分的 EV = 模型概率 × 赔率
3. 按以下两种策略各出一套方案：

**策略A — 覆盖优先：**
- 选模型概率最高的2个比分（不限方向）
- 目标：最大化命中率
- 适合保守型投注

**策略B — 方向一致：**
- 在预测方向（H/D/A）内选概率最高的2个比分
- 如果预测方向只有1个高概率比分，第2个从次优方向选（优先选平局比分）
- 目标：方向正确时回报更高
- 适合进攻型投注

**EV 亮点标注：**
- 如果某个比分 EV > 1.0（正期望），标注 `[+EV]`
- 这些比分即使单独看也值得关注

**输出格式：**

### Step 6: 写入投注看板

分析完成后，将本轮结论写入 `02_data/betting_rounds.json`。

**数据结构：** 每轮一个 entry，包含 `date`、`label`、`updatedAt`、`matches`（赔率+Edge）、`parlays`（串关方案）、`evHighlights`（+EV比分）。

**覆盖规则：** 如果该日期已有数据，替换（每轮只保留最后一次分析）。如果是新日期，追加。

写入后运行 `python3 generate_dashboard.py` 重新生成看板。

### 关键原则

- **HAD 串关需要所有场次都有 edge** — 否则串关只是在叠加劣势
- **低赔（<1.30）串关是陷阱** — 看似"稳"但一场冷门全部归零
- **CRS 串关核心是方向覆盖** — 在确定方向的基础上选2-3个比分
- **CRS 双比分是默认输出** — 每场选2个比分，同时展示覆盖优先和方向一致两种策略
- **标注 +EV 比分** — EV > 1.0 的比分单独高亮，即使不串关也值得关注
- **始终计算注数和金额** — 用户需要明确的费用
- **不强推串关** — 如果没有足够的 edge 场次，直接告诉用户不买更好
